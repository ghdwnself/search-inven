## Git 원격 저장소에 푸시하기

```powershell
# 현재 커밋 상태 확인
git status

# 원격 저장소에 푸시
git push origin main

# 또는 강제 푸시가 필요한 경우 (주의!)
# git push origin main --force
```

## 남은 작업

### 1. Credential 파일 정리 (권장)
```powershell
# 루트 폴더의 credential 파일을 server 폴더로 이동
Move-Item "warehouse-v1-480802-687b193253cf.json" "server/credentials.json"
```

### 2. 환경 변수 확인
`.env` 파일이 제대로 설정되어 있는지 확인:
```
GOOGLE_SHEET_ID=실제_시트_ID
GOOGLE_DRIVE_FOLDER_ID=실제_폴더_ID
GOOGLE_INVENTORY_SHEET_ID=실제_재고시트_ID (선택사항)
ADMIN_PASSWORD=관리자_비밀번호
```

### 3. 서버 테스트
```powershell
# 개발 서버 실행
npm run dev

# 브라우저에서 접속 테스트
# http://localhost:3000
```

## 주의사항 ⚠️

### Git에 절대 커밋하면 안 되는 파일
- ✅ `.env` - 이미 .gitignore에 포함됨
- ✅ `credentials.json` 또는 `*.json` (service account keys) - 이미 .gitignore에 포함됨
- ✅ `node_modules/` - 이미 .gitignore에 포함됨

### 만약 실수로 credential이 이미 원격 저장소에 올라갔다면
1. **즉시** Google Cloud Console에서 해당 서비스 계정 키 삭제
2. 새로운 키 생성
3. Git history에서 파일 제거 (복잡한 작업, 필요시 문의)
